### Etsy editörü otomasyonu — ✅ KÖK NEDEN ÇÖZÜLDÜ (21 Tem)

**Kök neden:** Etsy editörü React "controlled component" mimarisi kullanıyor. Bir input/textarea'nın DOM `.value`'suna **doğrudan atama** (`el.value = "..."`, event dispatch etmeden) yapılırsa DOM görsel olarak değişir ama React'in kendi state'i habersiz kalır → karakter sayacı eskisi gibi kalır, "You have no unsaved changes" banner'ı değişmez, **Publish changes disabled kalır**. Ekranda başarılı görünüp aslında hiçbir şey kaydedilmeye hazır olmaması — "sessiz doğrulama kilidi" budur. Canlı ilanda (`1077035558`) doğrulandı: naif `.value=` ataması sayacı ve Publish butonunu hiç etkilemedi.

**Doğrulanmış çözüm:** React'in value-tracker'ını atlatan native property setter + gerçek `input` event dispatch:
```js
const setter = Object.getOwnPropertyDescriptor(window.HTMLTextAreaElement.prototype /* input alanlar için HTMLInputElement.prototype */, 'value').set;
setter.call(element, yeniDeğer);
element.dispatchEvent(new Event('input', { bubbles: true }));
```
Bu yöntemle hem **title** hem **tag ekleme/silme** (native setter + input event + programatik `button.click()`) canlıda test edildi — sayaç, banner, Publish butonu hepsi doğru güncellendi. React tag input'unun DOM node'unu ekleme/silmede yeniden yaratmadığı da doğrulandı (referans eskimesi riski yok).

**Kalan risk:** Ardışık hızlı JS işlemlerinde React render'ının oturması için aralarda kısa bekleme + her adımdan sonra Publish butonunun `disabled` durumunu doğrulama (retry) önerilir — önceki "bazı etiketlerde sessizce düşüyor" gözlemi muhtemelen native-setter kullanılmadan yapılan denemelerden kaynaklanıyordu.

### Etsy tag ekleme
Yukarıdaki native-setter + input-event + programatik `button.click()` yöntemi güvenilir (21 Tem doğrulandı). Enter tuşu simülasyonu ayrıca test edilmedi, gerek kalmadı.
